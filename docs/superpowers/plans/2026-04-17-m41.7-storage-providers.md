# M41.7 — Storage Providers Parity Plan

**Goal:** Docstring + cross-language xref parity for the storage
provider subsystem across all three languages. All provider
implementations shipped in Milestone 39; this slice documents them
uniformly.

## Scope

- Protocol interfaces: StorageProvider, StorageGroup,
  StorageDataset + value class CompoundField.
- Concrete providers: Hdf5Provider, MemoryProvider.
- Registry: Python module-level functions vs Java/ObjC class.

## Pattern

All three languages carry the same cross-language xref format:
API status: Stable (Provisional per M39). plus an explicit
pointer at each of the other two language's idiomatic equivalent.
ObjC headers, Python docstrings (module + class), and Java Javadoc
(class-level + package-info) all updated.

## Tasks

1. Protocol interfaces + CompoundField.
2. Concrete providers + registry.
3. Slice commit.

## Test counts

Baseline: Python 180, Java 123, ObjC 867.
Expected: Python 182 (+2), Java 125 (+2), ObjC 867.

## Acceptance

- All three suites green.
- Cross-compat 8/8.
- Every provider-subsystem public class carries an API status line
  and cross-language xref.
