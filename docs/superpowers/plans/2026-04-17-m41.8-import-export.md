# M41.8 — Import/Export Parity Plan

**Goal:** Docstring + cross-language xref parity across the
Import/Export subsystem.

## Scope

10 subsystems:
- mzML Reader / Writer
- nmrML Reader / Writer
- Thermo Reader
- ISA Exporter
- Base64 helper (Python private, Java stdlib)
- CV Term Mapper
- Python ImportResult (Python-idiomatic helper; no ObjC/Java equivalent)

## Pattern

ObjC headers get xref blocks; Python module docstrings get
Cross-language equivalents footers; Java classes get Javadoc
extensions with .

All Import/Export code pre-dates this slice. No behavior changes.

## Known cross-language behavior divergence

CVTermMapper.precisionFor (Java) returns null for unknown
accessions, whereas ObjC +precisionForAccession: and Python
precision_for return Precision.FLOAT64 as a safe default.
Documented as a known difference for the slice 41.9 api-review
doc; NOT fixed here to avoid mixing behavior changes with docs
work.

## Test counts

Baseline: Python 182, Java 125, ObjC 867.
Expected: Python 184 (+2), Java 126 (+1 — no Java Base64 wrapper),
ObjC 867.

## Acceptance

- All three suites green.
- Cross-compat 8/8.
- Every Import/Export public class carries API status + xref.
