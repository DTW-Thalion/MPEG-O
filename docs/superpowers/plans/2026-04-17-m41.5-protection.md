## Task 2: `Verifier`

**Goal:** Create `VerificationStatus` enum + `Verifier` class in Python + Java. ObjC exposes a 4-state enum (`Valid`, `Invalid`, `NotSigned`, `Error`) and two class methods that wrap `TTIOSignatureManager` with status-aware return.

**Python** — new file `python/src/ttio/verifier.py`:

```python
"""``Verifier`` — high-level sign/verify status wrapper."""
from __future__ import annotations

from enum import IntEnum

from . import signatures


class VerificationStatus(IntEnum):
    """Sign-and-verify cycle outcome.

    Cross-language equivalents
    --------------------------
    Objective-C: ``TTIOVerificationStatus`` · Java:
    ``global.thalion.ttio.protection.Verifier.Status``.
    """

    VALID = 0
    INVALID = 1
    NOT_SIGNED = 2
    ERROR = 3


class Verifier:
    """High-level verification API.

    Collapses the three outcomes of a sign-and-verify cycle
    (valid / invalid / not-signed) into a single enum, plus an error
    fallback for I/O failures. Use this instead of
    :mod:`ttio.signatures` when you want to render a status to an
    end user.

    Notes
    -----
    API status: Stable.

    Cross-language equivalents
    --------------------------
    Objective-C: ``TTIOVerifier`` · Java:
    ``global.thalion.ttio.protection.Verifier``.
    """

    @staticmethod
    def verify_dataset(file_path: str, dataset_path: str, key: bytes) -> VerificationStatus:
        """Verify the signature on an HDF5 dataset.

        Parameters
        ----------
        file_path : str
            Path to the ``.tio`` file.
        dataset_path : str
            HDF5 dataset path inside the file.
        key : bytes
            32-byte HMAC key.

        Returns
        -------
        VerificationStatus
        """
        import h5py

        try:
            with h5py.File(file_path, "r") as f:
                if dataset_path not in f:
                    return VerificationStatus.ERROR
                ds = f[dataset_path]
                if signatures.SIGNATURE_ATTR not in ds.attrs:
                    return VerificationStatus.NOT_SIGNED
                # Delegate to signatures.verify_dataset (returns bool).
                try:
                    ok = signatures.verify_dataset(ds, key)
                except Exception:
                    return VerificationStatus.ERROR
                return VerificationStatus.VALID if ok else VerificationStatus.INVALID
        except OSError:
            return VerificationStatus.ERROR

    @staticmethod
    def verify_provenance_in_run(file_path: str, run_path: str, key: bytes) -> VerificationStatus:
        """Verify the provenance signature on a run group.

        Parameters
        ----------
        file_path : str
            Path to the ``.tio`` file.
        run_path : str
            HDF5 group path of the run (e.g. ``"/study/ms_runs/run_0001"``).
        key : bytes
            32-byte HMAC key.

        Returns
        -------
        VerificationStatus
        """
        import h5py

        try:
            with h5py.File(file_path, "r") as f:
                if run_path not in f:
                    return VerificationStatus.ERROR
                g = f[run_path]
                if signatures.PROVENANCE_SIGNATURE_ATTR not in g.attrs:
                    return VerificationStatus.NOT_SIGNED
                try:
                    ok = signatures.verify_provenance_in_run(g, key)
                except Exception:
                    return VerificationStatus.ERROR
                return VerificationStatus.VALID if ok else VerificationStatus.INVALID
        except OSError:
            return VerificationStatus.ERROR
```

```java
/* TTI-O Java Implementation / Copyright (C) 2026 DTW-Thalion / SPDX-License-Identifier: LGPL-3.0-or-later */
package global.thalion.ttio.protection;

/**
 * High-level verification API.
 *
 * <p>Collapses the three outcomes of a sign-and-verify cycle
 * (valid / invalid / not-signed) into a single enum, plus an
 * {@link Status#ERROR} fallback for I/O failures. Use this instead
 * of {@link SignatureManager} directly when you want to render a
 * status to an end user.</p>
 *
 * <p><b>API status:</b> Stable.</p>
 *
 * <p><b>Cross-language equivalents:</b> Objective-C
 * {@code TTIOVerifier}, Python
 * {@code ttio.verifier.Verifier}.</p>
 *
 * @since 0.6
 */
public final class Verifier {

    /** Sign-and-verify cycle outcome. */
    public enum Status {
        VALID,
        INVALID,
        NOT_SIGNED,
        ERROR
    }

    private Verifier() {}

    /**
     * Verify a signature string against data and key, returning a
     * three-state status.
     *
     * @param data      original bytes (never {@code null})
     * @param signature signature string, or {@code null} / empty for NOT_SIGNED
     * @param key       32-byte HMAC key
     * @return status
     */
    public static Status verify(byte[] data, String signature, byte[] key) {
        if (signature == null || signature.isEmpty()) return Status.NOT_SIGNED;
        try {
            return SignatureManager.verify(data, signature, key)
                ? Status.VALID : Status.INVALID;
        } catch (RuntimeException e) {
            return Status.ERROR;
        }
    }
}
```

**Simplified for Java:** the Python version opens the HDF5 file because `signatures.verify_dataset` in the current codebase likely takes an h5py object. Java's `SignatureManager.verify(data, signature, key)` takes raw bytes — simpler, no file I/O. Keep Java minimal. The Python version may also be simplified if its underlying `signatures` module takes bytes; investigate and pick the cleanest call pattern that matches the existing signature module's API. **When in doubt, keep the Verifier wrapper thin and just route bytes + signature + key**; the caller is responsible for extracting them from HDF5.

**Python simplification** — if `ttio.signatures` only has byte-level functions (`hmac_sha256`, `hmac_sha256_b64`) and no "verify a dataset by path" helper, change the Python Verifier to match Java's bytes-in interface:

```python
class Verifier:
    @staticmethod
    def verify(data: bytes, signature: str | None, key: bytes) -> VerificationStatus:
        if not signature:
            return VerificationStatus.NOT_SIGNED
        try:
            import base64, hmac
            payload = signature[3:] if signature.startswith("v2:") else signature
            expected = signatures.hmac_sha256(data, key)
            actual = base64.b64decode(payload)
            return VerificationStatus.VALID if hmac.compare_digest(expected, actual) else VerificationStatus.INVALID
        except Exception:
            return VerificationStatus.ERROR
```

This matches Java's shape and is idiomatic for the thin-wrapper purpose. Investigate the current `signatures.py` module and pick the simpler path.
