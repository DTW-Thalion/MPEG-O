# Phase 3: Fixture Regeneration + Cross-Lang Gate Revalidation

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Regenerate every committed `.tio` fixture under v1.0 defaults (after Phase 2a–2e dropped opt-flags + v1 codecs + dead enum slots), prune fixtures that test removed v1 codecs, fix stale references to the dropped `1.2.0` artifact name, and re-run the cross-language byte-equality gate.

**Architecture:** Phase 2c had to repair a small subset of fixtures inline (3 deleted, 6 regenerated with NAME_TOKENIZED_V2). Phase 3 closes that out: walk all 25 tracked `.tio` files, regenerate via the existing per-fixture scripts, sync Java's reference copies from ObjC, prune the two Java-only v1 NAME_TOKENIZED fixtures, refresh the two stale fixture READMEs, fix the three integration tests pointing at the pre-rebrand `ttio-1.2.0.jar`, then run the three test suites and the `pytest -m integration` cross-lang gate. Working directory is `~/TTI-O` inside WSL Ubuntu — invoke shell commands as `wsl -d Ubuntu -- bash -c '...'`.

**Tech Stack:** Python 3.12 (pytest), Java 21 (mvn 3.9, JUnit 5), Objective-C (gnustep-make + clang), HDF5 1.14, native libttio_rans (CMake/clang).
