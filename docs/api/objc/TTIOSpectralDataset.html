Root container for a TTI-O .tio file. Owns a top-level study/ group plus zero or more named MS acquisition runs, NMR spectrum collections, genomic runs, and the dataset-wide identifications, quantifications, provenance records, and an optional transition list.

Persistence is via -writeToFilePath:error: / +readFromFilePath:error: which open or create the underlying HDF5 file directly. The class also provides several +writeMinimalToPath: overloads — flat-buffer fast paths that bypass per-spectrum object construction for callers with already-flattened channel data (importers, numerical producers).

API status: Stable.

Cross-language equivalents:
Python: ttio.spectral_dataset.SpectralDataset
Java: global.thalion.ttio.SpectralDataset

decryptInPlaceAtPath: withKey: error: 

+ (BOOL) decryptInPlaceAtPath: (NSString*)path withKey: (NSData*)key error: (NSError**)error;
Persist-to-disk decrypt. Strips AES-256-GCM encryption from the .tio file at path : for every MS run with an encrypted intensity channel, writes the plaintext back as intensity_values and removes the encrypted siblings. Finally clears the root @encrypted attribute so -isEncrypted returns NO when the file is reopened.

Symmetric with -encryptWithKey:level:error: (which leaves the root attribute set). After this call the file is byte-compatible with the pre-encryption layout. The file must not be held open by another writer.

writeMinimalToPath: title: isaInvestigationId: mixedRuns: genomicRuns: identifications: quantifications: provenanceRecords: error: 

+ (BOOL) writeMinimalToPath: (NSString*)path title: (NSString*)title isaInvestigationId: (NSString*)isaId mixedRuns: (NSDictionary*)mixedRuns genomicRuns: (NSDictionary*)genomicRuns identifications: (NSArray*)identifications quantifications: (NSArray*)quantifications provenanceRecords: (NSArray*)provenance error: (NSError**)error;
Canonical mixed-dictionary write API. Accepts a single mixedRuns dict whose values may be either TTIOWrittenRun (MS) or TTIOWrittenGenomicRun (genomic); dispatches per-value via -isKindOfClass: to the right write path.

genomicRuns may also be supplied; a name appearing in BOTH dicts populates error rather than silently picking one. Unsupported value classes in mixedRuns also produce an error.

writeMinimalToPath: title: isaInvestigationId: msRuns: genomicRuns: identifications: quantifications: provenanceRecords: error: 

+ (BOOL) writeMinimalToPath: (NSString*)path title: (NSString*)title isaInvestigationId: (NSString*)isaId msRuns: (NSDictionary*)runs genomicRuns: (NSDictionary*)genomicRuns identifications: (NSArray*)identifications quantifications: (NSArray*)quantifications provenanceRecords: (NSArray*)provenance error: (NSError**)error;
Extended +writeMinimalToPath: accepting genomic runs alongside MS runs. Setting genomicRuns to a non-empty dictionary adds the opt_genomic feature flag. The shorter overload above delegates here with genomicRuns:nil.

writeMinimalToPath: title: isaInvestigationId: msRuns: identifications: quantifications: provenanceRecords: error: 

+ (BOOL) writeMinimalToPath: (NSString*)path title: (NSString*)title isaInvestigationId: (NSString*)isaId msRuns: (NSDictionary*)runs identifications: (NSArray*)identifications quantifications: (NSArray*)quantifications provenanceRecords: (NSArray*)provenance error: (NSError**)error;
Flat-buffer fast write path. Bypasses per-spectrum object construction and the channel-concat that -writeToFilePath:error: performs when given a TTIOAcquisitionRun of TTIOMassSpectrum objects. Callers that already have flat buffers (e.g. importers reading mzML in bulk, numerical producers) pass TTIOWrittenRun instances and skip both costs.

Writes the same on-disk layout as -writeToFilePath: , so readers do not distinguish files produced by the two paths.

closeFile 

- (BOOL) closeFile;
Releases the underlying HDF5 file handle. After this call any further lazy hyperslab reads on contained runs will fail. Required before calling -encryptWithKey:level:error: so the encryption manager can reopen the file read-write. Idempotent.
