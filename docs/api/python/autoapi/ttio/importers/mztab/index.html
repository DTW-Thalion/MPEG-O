ttio.importers.mztab¶

mzTab importer (v0.9 M60).

mzTab is a results-level interchange format from PSI. It is a tab-separated text file divided into sections by a one-line prefix:

  • MTD — metadata (key / value lines)

  • PRH / PRT — protein header / row (mzTab 1.0)

  • PEH / PEP — peptide header / row (mzTab 1.0)

  • PSH / PSM — PSM header / row (mzTab 1.0)

  • SMH / SML — small-molecule header / row (mzTab-M 2.0)

  • COM — comment line, ignored

  • anything else — treated as a malformed-row warning, skipped

Per HANDOFF binding decision 47 (mzTab version detection): MTD\tmzTab-version\t1.0 selects the proteomics dialect; MTD\tmzTab-version\t2.0.0-M selects the metabolomics dialect. The reader detects on the metadata line and dispatches accordingly.

Mapping into TTI-O records¶

mzTab section | TTI-O record | Notes |

|---------------|——————————|---------------------------------------------------------------| | PSM | Identification | run_name from spectra_ref; score from best search engine score | | PRT | Quantification | one record per protein_abundance_assay[N] column | | PEP | Feature | peptide-level feature (v0.12.0 M78) | | SML | Identification | metabolite annotation; run_name='metabolomics' placeholder | | SML abundance | Quantification | one record per abundance_study_variable[N] column | | SMF | Feature | small-molecule feature row (v0.12.0 M78) | | SME | Identification | small-molecule evidence row (v0.12.0 M78) | | MTD | ProvenanceRecord | software, search engine and ms_run locations |

to_ttio(path: str | pathlib.Path, *, title: str | None = None, isa_investigation_id: str = '', link_to: ttio.spectral_dataset.SpectralDataset | None = None) → pathlib.Path[source]¶

Write the imported records to an .tio container.

When link_to is None an identifications-only container is emitted (no spectra runs). When a SpectralDataset is supplied, its existing runs + identifications are merged with the mzTab records before writing — useful for splicing results back into the dataset that produced them. The merge preserves spectra_ref → run_name mappings recorded in the MTD ms_run[N]-location lines whenever the basename matches a run in the linked dataset.
