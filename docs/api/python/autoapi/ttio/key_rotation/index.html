ttio.key_rotation¶

envelope encryption + key rotation.

Mirrors the Objective-C TTIOKeyRotationManager. The wrapping primitive is AES-256-GCM (matching ttio.encryption). A DEK (data-encryption key) encrypts signal payloads; a KEK (key-encryption key) wraps the DEK. Rotation re-wraps the DEK under a new KEK without touching any other dataset, so it’s O(1) in file size.

On-disk layout under /protection/key_info/:

@kek_id (string) caller-supplied KEK identifier @kek_algorithm (string) “aes-256-gcm” @wrapped_at (string) ISO-8601 timestamp @key_history_json (string) JSON list of prior entries dek_wrapped (uint8[60]) 32 cipher + 12 IV + 16 tag

Usage:

import h5py, os
from ttio.key_rotation import (
    enable_envelope_encryption, rotate_key, unwrap_dek, key_history,
)

with h5py.File("x.tio", "w") as f:
    kek1 = os.urandom(32)
    dek = enable_envelope_encryption(f, kek1, kek_id="kek-1")
    # ... use dek to encrypt signal ...

with h5py.File("x.tio", "r+") as f:
    kek2 = os.urandom(32)
    rotate_key(f, old_kek=kek1, new_kek=kek2, new_kek_id="kek-2")

with h5py.File("x.tio", "r") as f:
    dek = unwrap_dek(f, kek=kek2)

ttio.key_rotation.enable_envelope_encryption(f: Any, kek: bytes, *, kek_id: str, algorithm: str = 'aes-256-gcm') → bytes[source]¶

Generate a fresh DEK, wrap it under kek, and persist key_info.

Returns the plaintext DEK so callers can use it to encrypt signal channels. Subsequent reads must unwrap via unwrap_dek() using the same KEK shape.

algorithm selects the wrap cipher suite:

  • "aes-256-gcm" — kek is a 32-byte symmetric key.

  • "ml-kem-1024" — kek is a 1568-byte ML-KEM public key. This writes the opt_pqc_preview feature flag onto the file ().

ttio.key_rotation.unwrap_dek(f: Any, kek: bytes, *, algorithm: str = 'aes-256-gcm') → bytes[source]¶

Unwrap and return the DEK using the given KEK.

Raises KeyRotationError (or a cryptography InvalidTag) when the KEK does not authenticate the wrapped blob. For algorithm="ml-kem-1024" pass the 3168-byte decapsulation private key.

ttio.key_rotation.rotate_key(f: Any, *, old_kek: bytes, new_kek: bytes, new_kek_id: str, algorithm: str = 'aes-256-gcm', new_algorithm: str | None = None) → None[source]¶

Re-wrap the DEK under new_kek and append the old entry to history.

Signal datasets are not touched, so the cost is O(1) in file size.

: new_algorithm may differ from algorithm to migrate a file from classical AES-256-GCM to ML-KEM-1024 (or back). When new_algorithm is omitted, the wrap stays on the same primitive.
