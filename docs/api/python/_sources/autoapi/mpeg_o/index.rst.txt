.. py:class:: Spectrum

   Generic multi-channel 1-D spectrum with per-scan metadata.

   Holds an ordered mapping of named :class:`SignalArray` objects and
   the coordinate axes that index them, plus per-scan metadata
   (index position, scan time, optional precursor info for tandem
   MS).

   Concrete subclasses (:class:`MassSpectrum`, :class:`NMRSpectrum`)
   add their own typed metadata.

   :param signal_arrays: Named signal arrays keyed by channel name
                         (``"mz"``, ``"intensity"``, ``"chemical_shift"``, ...).
   :type signal_arrays: dict[str, SignalArray]
   :param axes: Coordinate axes describing the signal arrays.
   :type axes: list[AxisDescriptor]
   :param index_position: Position in the parent :class:`~mpeg_o.acquisition_run.AcquisitionRun` (0-based).
   :type index_position: int, default 0
   :param scan_time_seconds: Scan time in seconds from run start.
   :type scan_time_seconds: float, default 0.0
   :param precursor_mz: Precursor m/z for tandem MS. ``0`` if not tandem.
   :type precursor_mz: float, default 0.0
   :param precursor_charge: Precursor charge state. ``0`` if unknown.
   :type precursor_charge: int, default 0

   .. rubric:: Notes

   API status: Stable.

   HDF5 representation: each spectrum is an HDF5 group whose
   immediate children are ``SignalArray`` sub-groups plus scalar
   attributes for the metadata fields.

.. py:class:: Identification

   A spectrum-level chemical-entity identification.

   Links a spectrum (by its 0-based index within an acquisition run)
   to a chemical entity with a confidence score and an evidence
   chain.

   :param run_name: Acquisition run that contains the spectrum.
   :type run_name: str
   :param spectrum_index: 0-based position within that run.
   :type spectrum_index: int
   :param chemical_entity: CHEBI accession or chemical formula.
   :type chemical_entity: str
   :param confidence_score: Score in ``[0.0, 1.0]``.
   :type confidence_score: float
   :param evidence_chain: Ordered list of free-form evidence strings (typically CV
                          accession references).

.. py:class:: AcquisitionRun

   Lazy view over one acquisition run inside an ``.mpgo`` file.

   Conforms to :class:`~mpeg_o.protocols.Indexable`,
   :class:`~mpeg_o.protocols.Streamable`, and
   :class:`~mpeg_o.protocols.Provenanceable`.
   :class:`~mpeg_o.protocols.Encryptable` conformance is delivered
   in slice 41.5 when the encryption manager subsystem lands.

   Spectrum access is zero-copy-aware: the spectrum index is
   pre-loaded into numpy arrays at open time but signal channels are
   sliced on demand, so random access to spectrum ``i`` touches only

.. py:class:: MSImage

   Mass-spectrometry imaging dataset: a ``width × height`` grid of
   pixels, each pixel a spectral profile of ``spectral_points`` values.

   :param width: Image grid dimensions in pixels.
   :type width: int, default 0
   :param height: Image grid dimensions in pixels.
   :type height: int, default 0
   :param spectral_points: Number of float64 values per pixel.
   :type spectral_points: int, default 0
   :param intensity: Rank-3 intensity cube of shape ``(height, width, spectral_points)``.
   :type intensity: numpy.ndarray, default empty rank-3 array
   :param pixel_size_x: Spatial pixel size (implementation-defined units).
   :type pixel_size_x: float, default 0.0
   :param pixel_size_y: Spatial pixel size (implementation-defined units).
   :type pixel_size_y: float, default 0.0
   :param scan_pattern: Scan pattern label (e.g. ``"raster"``, ``"random_access"``).
   :type scan_pattern: str, default ""
   :param tile_size: HDF5 chunk tile size for reads; 0 means native cube chunks.
   :type tile_size: int, default 0
   :param title: Dataset-level metadata. In Objective-C these are inherited
                 from ``MPGOSpectralDataset``; in Python they are composed
                 directly onto ``MSImage``.
   :type title: str, default ""
   :param isa_investigation_id: Dataset-level metadata. In Objective-C these are inherited
                                from ``MPGOSpectralDataset``; in Python they are composed
                                directly onto ``MSImage``.
   :type isa_investigation_id: str, default ""
   :param identifications: Dataset-level identifications (composed — see Notes).
   :type identifications: list, default []
   :param quantifications: Dataset-level quantifications (composed).
   :type quantifications: list, default []
   :param provenance_records: Dataset-level provenance records (composed).
   :type provenance_records: list, default []

   .. rubric:: Notes

   API status: Stable.

   **Composition vs inheritance.** In Objective-C ``MPGOMSImage``
   inherits from ``MPGOSpectralDataset`` so dataset-level fields come
   for free. In Python, ``SpectralDataset`` is a file-handle wrapper
   whose lifecycle does not map cleanly to an MSImage subclass;
   composition is used here (the five dataset-level fields live on
   ``MSImage`` directly). This is a known stylistic difference
   between the language implementations, recorded in
   :doc:`/api-review-v0.6`.

   Cross-language equivalents

   ``com.dtwthalion.mpgo.MSImage``.
