.. py:class:: ReferenceImport

   Reference-FASTA contents staged for embedding into a ``.tio`` container.

   :param uri: Reference identifier (e.g. ``"GRCh38.p14"``). Becomes the
               sub-group name under ``/study/references/<uri>/``.
   :type uri: str
   :param chromosomes: Chromosome names in FASTA file order. The on-disk MD5 is
                       order-invariant (see :func:`compute_reference_md5`).
   :type chromosomes: list[str]
   :param sequences: Per-chromosome sequence bytes, one entry per chromosome,
                     case-preserved. Newlines and whitespace stripped.
   :type sequences: list[bytes]
   :param md5: 16-byte content MD5 (see :func:`compute_reference_md5`). If
               omitted, computed from ``chromosomes`` + ``sequences``.
   :type md5: bytes

   .. rubric:: Notes

   A round-trip FASTA -> :class:`FastaReader` -> ``ReferenceImport``
   -> :class:`FastaWriter` -> FASTA preserves byte content
   (including soft-masking via lowercase). The Reference Resolver's
   upper-casing for REF_DIFF_V2 is a separate normalisation that
   happens at decode time.

   .. py:method:: write_to_dataset(dataset: ttio.spectral_dataset.SpectralDataset, *, overwrite: bool = False) -> None

      Embed this reference at ``/study/references/<uri>/``
      inside ``dataset``'s open HDF5 file.

      Layout (cross-language byte-equal):

      ``/study/references/<uri>/``
        attr ``md5``: 32-character lowercase hex string
        attr ``total_bases``: int64 sum of sequence lengths
        ``chromosomes/<name>/data``  uint8 dataset of the
            chromosome's sequence bytes (case-preserving, no
            newlines).

      :param dataset: Open dataset (writable HDF5 backing).
      :type dataset: SpectralDataset
      :param overwrite: If ``True``, replace any existing reference under the same
                        URI; if ``False``, raise on collision.
      :type overwrite: bool

      :raises FileExistsError: If a reference with the same ``uri`` is already embedded
          and ``overwrite`` is ``False``.
      :raises RuntimeError: If ``dataset``'s storage backend doesn't expose an open
          HDF5 group (only HDF5-backed datasets support reference
          embed).



.. py:function:: compute_reference_md5(chromosomes: list[str], sequences: list[bytes]) -> bytes

   Compute the canonical MD5 over a reference's chromosome set.

   Algorithm (cross-language byte-exact):
       for each (name, seq) in sorted(zip(chromosomes, sequences),
                                      key=name):
           h.update(name_utf8 + b"\n")
           h.update(seq)
           h.update(b"\n")
       return h.digest()

   Sorting by name makes the MD5 invariant to FASTA record order.
   Names are encoded UTF-8; sequences are passed through verbatim
   (case-preserving). The trailing ``\n`` separators are pure
   framing and never appear inside sequence bytes (FASTA forbids
   embedded newlines per record).

   :returns: *bytes* -- 16-byte MD5 digest.


.. py:class:: ReferenceResolver(h5_file: h5py.File, external_reference_path: pathlib.Path | None = None)

   Resolve a reference chromosome sequence for REF_DIFF decode.

   Args:
       h5_file: open ``h5py.File`` handle in read mode. The resolver
           looks for ``/study/references/<uri>/`` as the primary source.
       external_reference_path: optional explicit path to a FASTA file.
           If unset, the ``REF_PATH`` environment variable is consulted.

      Return the chromosome's reference sequence as uppercase ACGTN bytes.

      Raises:
          RefMissingError: when the reference can't be found or its
              MD5 doesn't match.



.. py:exception:: RefMissingError

   Bases: :py:obj:`RuntimeError`


   Raised when a reference required for REF_DIFF decode cannot be resolved.

   Per M93 design spec Q5c: hard error rather than partial decode.
   Genomic data integrity is non-negotiable.

   Initialize self.  See help(type(self)) for accurate signature.
