# TTI-O M82 — Genomic Data Model

> **Status:** shipped (v0.11, M82.1–M82.4). All three reference
> implementations (Python, ObjC, Java) write and read the same
> on-disk shape; the 9-cell cross-language conformance matrix
> (`python/tests/validation/test_m82_3x3_matrix.py`) gates every CI
> run.

This guide is for **downstream users** integrating with the
genomic side of a `.tio` file. It covers what the four new types
are, how they map onto the existing MS pattern, the on-disk layout
under `/study/genomic_runs/`, and a minimal write+read snippet in
each language. For the architectural rationale (where the API can
stay polymorphic with MS code and where it must diverge) see the
"Genomic abstraction-layer divergence" section in `ARCHITECTURE.md`.

---

## What's in M82

M82 introduces a parallel run-and-element hierarchy alongside the
spectrum-based classes. The four new public types are:

| Type | Role | MS analogue |
|---|---|---|
| `GenomicRun` | Lazy view over one aligned-read run; `Indexable<AlignedRead>` + `Streamable<AlignedRead>` + `AutoCloseable` | `AcquisitionRun` |
| `AlignedRead` | Per-read value: `read_name`, `chromosome`, `position`, `mapping_quality`, `cigar`, `sequence`, `qualities`, `flags`, mate-pair info | `MassSpectrum` etc. |
| `GenomicIndex` | Parallel-array index: offsets, lengths, chromosomes, positions, mapping qualities, flags. Supports region / unmapped / flag queries | `SpectrumIndex` |
| `WrittenGenomicRun` | Pure write-side container; passed to `SpectralDataset.write_minimal` | `WrittenRun` |

`SpectralDataset` itself was extended with a `genomic_runs` map
that lives **alongside** `ms_runs`. A single `.tio` file may hold
any combination — MS-only, genomic-only, or both (multi-omics).
Pre-M82 readers (without the `OPT_GENOMIC` feature flag) see
`genomic_runs == {}` and continue to work unchanged.

### What's deliberately out of scope

- **All genomic compression codecs.** M82 stores genomic data
  uncompressed: one ASCII byte per base in `signal_channels/sequences`,
  raw Phred bytes in `signal_channels/qualities`, VL_STRING in the
  `read_names` and `cigars` compounds. M79 reserved five codec enum
  slots (`RANS_ORDER0=4`, `RANS_ORDER1=5`, `BASE_PACK=6`,
  `QUALITY_BINNED=7`, `NAME_TOKENIZED=8` — all CRAM-3.0-era) but
  **none of them are implemented in any of the three languages**.
  CRAM 3.1's newer codecs (rANS-Nx16, fqzcomp, adaptive
  arithmetic) are **not even reserved**; they would need additional
  enum slots. Tracked under "Genomic codec milestone" in WORKPLAN.

  *Practical impact:* a 100-read × 150 bp file is ~61 KB
  uncompressed; the equivalent BAM/CRAM is ~6–10 KB. For 10 M
  reads the gap becomes ~3 GB vs ~300 MB. M82 is functionally
  complete but not deployable for production-scale sequencing
  data until the codec milestone ships.
- **Secondary / supplementary alignment representation.** Only the
  primary alignment record per read is stored. Secondary alignments
  (SAM flag 0x100) and supplementary alignments (0x800) can be
  written and surface via `flags`, but there is no separate
  secondary-alignment table.
- **Multi-omics joins.** Holding an MS run and a genomic run in the
  same file is supported; helpers that *correlate* them across
  modalities are a separate analysis milestone.
- **MS runs via memory:// / sqlite:// / zarr://.** Genomic runs
  work on all four providers; MS runs through non-HDF5 providers
  remain on the M82.2/M82.3/M82.4 deferred list.

## On-disk layout

A genomic run lives at `/study/genomic_runs/<run-name>/` with the
following children. All datasets use the standard TTI-O storage
substrate (`StorageGroup` / `StorageDataset`) and respect the
provider's chunking / compression policy.

```
/study/genomic_runs/<name>/
├── @acquisition_mode    UINT8  attr      (AcquisitionMode enum)
├── @modality            UTF-8  attr      ("genomic_sequencing")
├── @reference_uri       UTF-8  attr      e.g. "GRCh38.p14"
├── @platform            UTF-8  attr      e.g. "ILLUMINA"
├── @sample_name         UTF-8  attr      e.g. "NA12878"
├── genomic_index/
│   ├── offsets          UINT64[N]        per-read byte offset
│   ├── lengths          UINT32[N]        per-read length
│   ├── chromosomes      VL_STRING[N]     per-read chromosome
│   ├── positions        INT64[N]         0-based mapping position
│   ├── mapping_qualities UINT8[N]
│   └── flags            UINT32[N]        SAM flag bits
└── signal_channels/
    ├── sequences        UINT8[total]     concatenated bases (ACGT)
    ├── qualities        UINT8[total]     concatenated Phred
    ├── cigars           compound { string }
    ├── read_names       compound { string }
    └── mate_info        compound { string mate_chrom; int64 mate_pos; int32 template_len }
```

`total = sum(lengths)`. The compound rows are aligned 1-to-1 with
`genomic_index/offsets`. VL_STRING in the compounds is **byte-
identical** across Python (h5py), ObjC (libhdf5), and Java
(JHI5 + `sun.misc.Unsafe` for the read-back path — see the
M82.4 CHANGELOG entry).

When any genomic run is present, the file's `feature_flags`
include `opt_genomic` and `format_version` is bumped to `"1.4"`.

## Minimal write + read snippets

The example below writes a 3-read genomic run, opens the file, and
prints the first read. The shape is identical across languages
(differences are syntactic, not structural).

### Python

```python
import numpy as np
from ttio import SpectralDataset
from ttio.written_genomic_run import WrittenGenomicRun

run = WrittenGenomicRun(
    acquisition_mode=7,                       # GENOMIC_WGS
    reference_uri="GRCh38.p14",
    platform="ILLUMINA",
    sample_name="NA12878",
    positions=np.array([100, 200, 300], dtype=np.int64),
    mapping_qualities=np.full(3, 60, dtype=np.uint8),
    flags=np.zeros(3, dtype=np.uint32),
    sequences=np.frombuffer(b"ACGT" * 6, dtype=np.uint8),  # 24 bases
    qualities=np.frombuffer(bytes([30] * 24), dtype=np.uint8),
    offsets=np.array([0, 8, 16], dtype=np.uint64),
    lengths=np.array([8, 8, 8], dtype=np.uint32),
    cigars=["8M", "8M", "8M"],
    read_names=["r0", "r1", "r2"],
    mate_chromosomes=["", "", ""],
    mate_positions=np.full(3, -1, dtype=np.int64),
    template_lengths=np.zeros(3, dtype=np.int32),
    chromosomes=["chr1", "chr1", "chr2"],
)

SpectralDataset.write_minimal(
    "out.tio", title="demo", isa_investigation_id="ISA-DEMO",
    runs={}, genomic_runs={"genomic_0001": run},
)

with SpectralDataset.open("out.tio") as ds:
    gr = ds.genomic_runs["genomic_0001"]
    print(f"{len(gr)} reads on {gr.reference_uri}")
    r0 = gr[0]
    print(r0.read_name, r0.chromosome, r0.position, r0.cigar, r0.sequence)
```

### Java

```java
import global.thalion.ttio.*;
import global.thalion.ttio.Enums.*;
import global.thalion.ttio.genomics.*;
import java.util.List;

WrittenGenomicRun run = new WrittenGenomicRun(
    AcquisitionMode.GENOMIC_WGS,
    "GRCh38.p14", "ILLUMINA", "NA12878",
    new long[]{100, 200, 300},
    new byte[]{60, 60, 60},
    new int[]{0, 0, 0},
    "ACGTACGTACGTACGTACGTACGT".getBytes(),
    new byte[24],   // qualities (filled with 0 for brevity)
    new long[]{0, 8, 16},
    new int[]{8, 8, 8},
    List.of("8M", "8M", "8M"),
    List.of("r0", "r1", "r2"),
    List.of("", "", ""),
    new long[]{-1, -1, -1},
    new int[]{0, 0, 0},
    List.of("chr1", "chr1", "chr2"),
    Compression.ZLIB);

SpectralDataset.create(
    "out.tio", "demo", "ISA-DEMO",
    List.of(), List.of(run),
    List.of(), List.of(), List.of(),
    FeatureFlags.defaultCurrent()).close();

try (SpectralDataset ds = SpectralDataset.open("out.tio")) {
    GenomicRun gr = ds.genomicRuns().get("genomic_0001");
    System.out.println(gr.readCount() + " reads on " + gr.referenceUri());
    AlignedRead r0 = gr.readAt(0);
    System.out.println(r0.readName() + " " + r0.chromosome()
        + " " + r0.position() + " " + r0.cigar() + " " + r0.sequence());
}
```

### Objective-C

```objc
#import "Dataset/TTIOSpectralDataset.h"
#import "Genomics/TTIOWrittenGenomicRun.h"
#import "Genomics/TTIOGenomicRun.h"
#import "Genomics/TTIOAlignedRead.h"

NSMutableData *positions = [NSMutableData dataWithLength:3 * sizeof(int64_t)];
((int64_t *)positions.mutableBytes)[0] = 100;
((int64_t *)positions.mutableBytes)[1] = 200;
((int64_t *)positions.mutableBytes)[2] = 300;
NSMutableData *mapqs    = [NSMutableData dataWithLength:3];
memset(mapqs.mutableBytes, 60, 3);
NSMutableData *flags    = [NSMutableData dataWithLength:3 * sizeof(uint32_t)];
NSData       *sequences = [@"ACGTACGTACGTACGTACGTACGT" dataUsingEncoding:NSASCIIStringEncoding];
NSMutableData *quals    = [NSMutableData dataWithLength:24];
NSMutableData *offsets  = [NSMutableData dataWithLength:3 * sizeof(uint64_t)];
((uint64_t *)offsets.mutableBytes)[0] = 0;
((uint64_t *)offsets.mutableBytes)[1] = 8;
((uint64_t *)offsets.mutableBytes)[2] = 16;
NSMutableData *lengths  = [NSMutableData dataWithLength:3 * sizeof(uint32_t)];
((uint32_t *)lengths.mutableBytes)[0] = 8;
((uint32_t *)lengths.mutableBytes)[1] = 8;
((uint32_t *)lengths.mutableBytes)[2] = 8;
NSMutableData *matePos  = [NSMutableData dataWithLength:3 * sizeof(int64_t)];
((int64_t *)matePos.mutableBytes)[0] = -1;
((int64_t *)matePos.mutableBytes)[1] = -1;
((int64_t *)matePos.mutableBytes)[2] = -1;
NSMutableData *tlens    = [NSMutableData dataWithLength:3 * sizeof(int32_t)];

TTIOWrittenGenomicRun *run = [[TTIOWrittenGenomicRun alloc]
    initWithAcquisitionMode:TTIOAcquisitionModeGenomicWGS
               referenceUri:@"GRCh38.p14"
                   platform:@"ILLUMINA"
                 sampleName:@"NA12878"
                  positions:positions
           mappingQualities:mapqs
                      flags:flags
                  sequences:sequences
                  qualities:quals
                    offsets:offsets
                    lengths:lengths
                     cigars:@[@"8M", @"8M", @"8M"]
                  readNames:@[@"r0", @"r1", @"r2"]
            mateChromosomes:@[@"", @"", @""]
              matePositions:matePos
            templateLengths:tlens
                chromosomes:@[@"chr1", @"chr1", @"chr2"]
          signalCompression:TTIOCompressionZlib];

NSError *err = nil;
[TTIOSpectralDataset writeMinimalToPath:@"out.tio"
                                   title:@"demo"
                      isaInvestigationId:@"ISA-DEMO"
                                  msRuns:@{}
                             genomicRuns:@{@"genomic_0001": run}
                         identifications:nil
                         quantifications:nil
                       provenanceRecords:nil
                                   error:&err];

TTIOSpectralDataset *ds = [TTIOSpectralDataset readFromFilePath:@"out.tio"
                                                          error:&err];
TTIOGenomicRun *gr = ds.genomicRuns[@"genomic_0001"];
TTIOAlignedRead *r0 = [gr readAtIndex:0 error:&err];
NSLog(@"%@ %@ %lld %@ %@", r0.readName, r0.chromosome,
      (long long)r0.position, r0.cigar, r0.sequence);
```

## Querying

`GenomicIndex` exposes three query helpers that don't materialise
reads:

- `indices_for_region(chromosome, start, end)` — returns the
  indices of reads on `chromosome` whose `position` is in
  `[start, end)`.
- `indices_for_unmapped()` — reads with SAM flag 0x4 set.
- `indices_for_flag(mask)` — reads with `flags & mask != 0`.

For full reads materialise them with `gr[i]` (Python),
`gr.readAt(i)` (Java), or `[gr readAtIndex:i error:&err]` (ObjC).

```python
gr = ds.genomic_runs["genomic_0001"]
chr1_window = gr.index.indices_for_region("chr1", 10_000, 20_000)
reads = [gr[i] for i in chr1_window]
```

## Cross-language fixture

The committed fixture
`python/tests/fixtures/genomic/m82_100reads.tio` (61 KB) is the
canonical reference shape for cross-language conformance. It is
produced by `python/tests/fixtures/genomic/generate.py` and read
by all three languages' test suites. If you are implementing a
fourth-language reader, this is the file to validate against.

The 9-cell matrix
(`python/tests/validation/test_m82_3x3_matrix.py`) re-generates
deterministic 100-read fixtures from each language's writer
(`TtioWriteGenomicFixture` in Java + ObjC,
`_write_python_fixture` helper in the matrix test) on every CI
run, so writer drift in any language is caught immediately.
