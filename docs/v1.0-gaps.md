# v1.0 Gap Audit — Exporter + Importer Completeness

Produced 2026-04-19 as follow-up to the M64 xfail markers. This
document inventories every exporter/importer defect currently known,
graded for v1.0 planning.

**Update 2026-04-19 (post-fix commit):** mzML XSD validation and
ISA-Tab isatools validation now pass. The nmrML wrapper sections also
now validate; only `<spectrum1D>` content-model remains xfailed, now
with a more precise reason (data-model bridge needed, not just writer
changes). See "Fix status" table below.

**All defects are shared across the three language implementations
(ObjC / Python / Java).** The three writers are maintained together
with intentional byte-parity; fixing a defect means coordinating the
change across all three source trees.

## Exporters

### mzML writer — 1 high-severity defect + 3 minor

**Shared source locations:**
`python/src/mpeg_o/exporters/mzml.py`,
`objc/Source/Export/MPGOMzMLWriter.m`,
`java/src/main/java/com/dtwthalion/mpgo/exporters/MzMLWriter.java`

| # | Defect | Severity | Fix complexity |
|---|--------|----------|----------------|
| 1 | `<precursor>` missing required `<activation>` child for MS2 spectra | **HIGH** — fails PSI XSD validation; breaks pyteomics strict mode on some tools | Low — add a default `<cvParam accession="MS:1000133" name="collision-induced dissociation"/>` inside a new `<activation>` block. Would be better to preserve the real method from the source mzML, which requires a data-model extension (see item 2). |
| 2 | `activation_method` + `isolation_window` not preserved in the data model | MED — cannot distinguish CID / HCD / ETD / UVPD on round-trip | Medium — add new fields to `MPGOMassSpectrum`, `ImportResult.ImportedSpectrum`, and the `spectrum_index` compound schema. Reader must populate them; writer must emit them. |
| 3 | `<instrumentConfiguration>` emitted empty | LOW — validates but tools can't tell what instrument produced the run | Medium — we already store `MPGOInstrumentConfig`; mapping its fields to PSI cvParams (`MS:1000483` manufacturer, `MS:1000529` model) is straightforward. |
| 4 | `<softwareList>` + `<dataProcessingList>` emit one placeholder entry each | LOW — reviewers sometimes flag missing provenance chain | Low — we have `ProvenanceRecord` in `AcquisitionRun`; emit each as a `<processingMethod>`. |

**Default CI behavior:** xfailed strict=False under
`@requires_network test_mzml_export_validates_against_psi_xsd`.
Well-formed-XML baseline passes; pyteomics + pymzml can parse the
output despite the XSD violation (both readers are lenient about
the missing `<activation>`).

### nmrML writer — 2 high-severity defects

**Shared source locations:**
`python/src/mpeg_o/exporters/nmrml.py`,
`objc/Source/Export/MPGONmrMLWriter.m`,
`java/src/main/java/com/dtwthalion/mpgo/exporters/NmrMLWriter.java`

| # | Defect | Severity | Fix complexity |
|---|--------|----------|----------------|
| 1 | `<nmrML>` root missing required `version="1.1"` attribute | **HIGH** — fails nmrML XSD | Trivial — one-line fix in all three writers. |
| 2 | `<fileDescription>` section missing; `<acquisition>` emitted directly after `<cvList>` | **HIGH** — XSD requires `cvList, fileDescription, [optional], acquisition, spectrumList` in that order | Low — emit a minimal `<fileDescription>` block with `<fileContent>` + generic `cvParam`. |
| 3 | Spectrometer frequency attribute is always 0 — the writer's `freq_mhz` variable has no source | MED — NMR round-trip loses instrument calibration | Low — plumb `AcquisitionRun.spectrometer_frequency_mhz` through to the writer (field exists on the run, just not threaded into `spectrum_to_bytes`). |

**Default CI behavior:** xfailed strict=False under
`test_nmrml_export_validates_against_xsd`. Well-formed-XML baseline
passes.

### ISA-Tab exporter — 7 missing required sections

**Shared source locations:**
`python/src/mpeg_o/exporters/isa.py`,
`objc/Source/Export/MPGOISAExporter.m`,
`java/src/main/java/com/dtwthalion/mpgo/exporters/ISAExporter.java`

Current investigation-file sections emitted:

```
✓ ONTOLOGY SOURCE REFERENCE
✓ INVESTIGATION
✗ INVESTIGATION PUBLICATIONS          <-- isatools rejects here
✗ INVESTIGATION CONTACTS
✓ STUDY
✗ STUDY DESIGN DESCRIPTORS
✗ STUDY PUBLICATIONS
✗ STUDY FACTORS
✓ STUDY ASSAYS
✗ STUDY PROTOCOLS
✗ STUDY CONTACTS
```

**Severity:** HIGH — the isatools validator halts at the first
missing required section. 7 of 11 required sections absent; fixing
just `INVESTIGATION PUBLICATIONS` would reveal the next error, not
pass the full validator.

**Fix complexity:** Medium. The seven missing sections are all
optional-content / required-header blocks — the header row must be
present even when every data row is blank. Per the ISA-Tab 1.0 spec
each section has a known column set. One afternoon of template
wiring per language; no data-model changes needed because the
dataset already has enough context to emit empty-but-valid tables.

**Default CI behavior:** xfailed strict=False under
`test_isatab_export_validates_with_isatools`.

## Importer coverage gaps

### mzTab importer — documented skips

`python/src/mpeg_o/importers/mztab.py`:

- **PEH/PEP** (peptide-level quantification) — intentionally deferred
- **SFH/SMF/SEH/SME** (small-molecule features + evidence) — deferred

Both are per-row data that currently has no MPEG-O target record
(we'd need a `Feature` value class alongside `Identification` and
`Quantification`). Marked as v1.0+ in the module docstring; not a
defect for v0.9 scope.

### mzML importer — does not preserve activation/isolation

Same root cause as mzML exporter defect #2. The reader parses
cvParams inside `<precursor>` but discards them — only `precursorMz`
and `precursorCharge` make it into `ImportedSpectrum`. Round-tripping
MS2 mzML through MPEG-O loses fragmentation method and isolation
window, which several downstream tools rely on.

Fix is co-located with exporter defect #2: extend the data model so
reader preserves and writer emits.

## Triage for v1.0 planning

**Must-fix for v1.0** (3 writer defects + 1 reader data model
extension):

1. mzML: add `<activation>` + `<instrumentConfiguration>` content
2. nmrML: add `version` attr + `<fileDescription>` + order fix
3. ISA-Tab: add the seven missing investigation-file sections
4. `activation_method` + `isolation_window` data-model extension
   (reader + writer in all 3 langs + spectrum_index schema bump)

**Nice-to-have for v1.0** (2 missing exporters):

5. imzML writer (so we can read → MPEG-O → export back to imzML)
6. mzTab writer (so we can publish identifications + quantifications)

**Deferred further** (scope-expansion, not defects):

- mzTab PEH/PEP + SFH/SMF/SEH/SME sections — needs new `Feature`
  value class
- mzML `<softwareList>` / `<dataProcessingList>` content — only
  needed if users want full provenance chain in the export

## What this does NOT mean

The v0.9.0 release is not broken by any of these. Every current
user-facing API works; the defects surface only when external
validators (isatools, strict PSI XSD, pyteomics tutorial mode) are
applied to the outputs. The three xfailed M64 tests make the
defects visible in CI output without failing the build — intentional,
so v1.0 planning can see the error logs without pretending
everything is perfect.
