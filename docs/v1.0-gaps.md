# v1.0 Gap Audit — Exporter + Importer Completeness

Produced 2026-04-19 as follow-up to the M64 xfail markers. This
document inventories every exporter/importer defect currently known,
graded for v1.0 planning.

**Update 2026-04-20 (v0.10.0 released):** The v0.10.0 release shipped
the streaming transport layer (M66–M72) and v1.0 per-Access-Unit
encryption across all three languages — these were the other major
"v1.0-blocker" items from earlier planning and are no longer gating.
The exporter/importer defects inventoried below remain for the v1.0
exporter-fidelity cut; most are format-model extensions rather than
writer tweaks.

**Update 2026-04-21 (v0.11.0 released):** M73 added Raman and IR as
first-class modalities with a JCAMP-DX 5.01 AFFN reader + writer in
all three languages (see `docs/vendor-formats.md`). The importer /
exporter defects below are unchanged; the format-model extension
from M73 did not require any change to the mzML / nmrML / ISA-Tab
writers.

**Update 2026-04-21 (v0.11.1 — post-audit):** Re-checked the
exporter surface directly — `imzML` and `mzTab` writers shipped in
v0.9.1 (commits `ff0f201` and `3c67ba9`) across all three languages;
the triage table below has been corrected. The only remaining
"must-fix for v1.0" item is the `activation_method` /
`isolation_window` data-model extension, now tracked as v0.12.0 M74
in `WORKPLAN.md`.

**Update 2026-04-22 (M74 shipped):** `activation_method` +
`isolation_window` landed across all three languages as five
sequential slices (`beb2bc7`, `736ecef`, `9340007`, `c502d68`,
`e96105f`). `MassSpectrum` carries the new fields, `spectrum_index`
compound carries four parallel columns, the mzML reader preserves
CID / HCD / ETD / UVPD / ECD / EThcD + isolation window, and the
writer emits both `<activation>` and `<isolationWindow>` in the
XSD-required order. Format version bumps to `1.3` on files that
actually carry the columns; the `opt_ms2_activation_detail` feature
flag advertises the extension so pre-M74 readers can still open the
file and ignore the extra columns. **The "Must-fix for v1.0" list
is now empty.**

**Update 2026-04-21 (v0.11.1 released):** M73.1 closed three of the
v0.11 deferred items:

- JCAMP-DX PAC / SQZ / DIF / DUP **reader** in all three languages
  (writers remain AFFN-only; see `docs/vendor-formats.md`).
- `UVVisSpectrum` class with JCAMP-DX `UV/VIS SPECTRUM` dispatch
  and writer.
- `TwoDimensionalCorrelationSpectrum` (Noda 2D-COS) with rank-2
  synchronous + asynchronous matrices, gated behind
  `opt_native_2d_cos`.

2-D JCAMP-DX `##NTUPLES=` / `##PAGE=` remains explicitly out of
scope (Raman/IR/UV-Vis cubes are stored as native HDF5 groups).

**Update 2026-04-19 (post-fix commit):** mzML XSD validation and
ISA-Tab isatools validation now pass. The nmrML wrapper sections also
now validate; only `<spectrum1D>` content-model remains xfailed, now
with a more precise reason (data-model bridge needed, not just writer
changes). See "Fix status" table below.

**All defects are shared across the three language implementations
(ObjC / Python / Java).** The three writers are maintained together
with intentional byte-parity; fixing a defect means coordinating the
change across all three source trees.

### ISA-Tab exporter — 7 missing required sections

**Shared source locations:**
`python/src/mpeg_o/exporters/isa.py`,
`objc/Source/Export/MPGOISAExporter.m`,
`java/src/main/java/com/dtwthalion/mpgo/exporters/ISAExporter.java`

Current investigation-file sections emitted:

```
✓ ONTOLOGY SOURCE REFERENCE
✓ INVESTIGATION
✗ INVESTIGATION PUBLICATIONS          <-- isatools rejects here
✗ INVESTIGATION CONTACTS
✓ STUDY
✗ STUDY DESIGN DESCRIPTORS
✗ STUDY PUBLICATIONS
✗ STUDY FACTORS
✓ STUDY ASSAYS
✗ STUDY PROTOCOLS
✗ STUDY CONTACTS
```

**Severity:** HIGH — the isatools validator halts at the first
missing required section. 7 of 11 required sections absent; fixing
just `INVESTIGATION PUBLICATIONS` would reveal the next error, not
pass the full validator.

**Fix complexity:** Medium. The seven missing sections are all
optional-content / required-header blocks — the header row must be
present even when every data row is blank. Per the ISA-Tab 1.0 spec
each section has a known column set. One afternoon of template
wiring per language; no data-model changes needed because the
dataset already has enough context to emit empty-but-valid tables.

**Default CI behavior:** xfailed strict=False under
`test_isatab_export_validates_with_isatools`.

## Importer coverage gaps

### mzTab importer — documented skips ✅ shipped

**Resolved v0.12.0 M78 (2026-04-23).** A new `Feature` value class
lands in all three languages beside `Identification` and
`Quantification`, carrying `feature_id`, `run_name`,
`chemical_entity`, `retention_time_seconds`, `exp_mass_to_charge`,
`charge`, `adduct_ion`, `abundances`, and `evidence_refs`. The mzTab
reader parses PEH/PEP rows (peptide-level quantification, 1.0
dialect) and SFH/SMF rows (small-molecule features, 2.0.0-M
dialect); SEH/SME parsing then back-fills the feature's
`chemical_entity` from `SME_ID` placeholder to
`database_identifier` / `chemical_name` / `chemical_formula`. The
mzTab writer emits those sections with an inverse rank↔confidence
mapping (rank N ↔ confidence 1/N). SEH/SME emission is gated on
features being present so plain-SML metabolomics round-trips stay
byte-identical with pre-M78 output.

### mzML importer — does not preserve activation/isolation ✅ shipped

**Resolved v0.12.0 M74 (2026-04-22).** The reader now maps the PSI-MS
activation cvParams (MS:1000133 CID, MS:1000422 HCD, MS:1000598 ETD,
MS:1000250 ECD, MS:1003246 UVPD, MS:1003181 EThcD) to the
`ActivationMethod` enum and reconstructs an `IsolationWindow` from
MS:1000827/828/829. Round-tripping MS2 mzML through MPEG-O now
preserves both fragmentation method and isolation window on a single
MassSpectrum instance, and both survive the HDF5 write/read cycle via
the four new optional parallel columns on `spectrum_index`.

## What this does NOT mean

The v0.9.0 release is not broken by any of these. Every current
user-facing API works; the defects surface only when external
validators (isatools, strict PSI XSD, pyteomics tutorial mode) are
applied to the outputs. The three xfailed M64 tests make the
defects visible in CI output without failing the build — intentional,
so v1.0 planning can see the error logs without pretending
everything is perfect.
